相信coder都或多或少见到过这样的神奇代码。这行代码能在WEB上运行，而且输出的内容各式各样。今天就用这篇文章来探秘下到底是怎么个野路子。

jsfuck#

上述代码其实是使用了Function定义匿名函数，然后立即执行

那么怎么从简单的[ ] + ! 之类的符号获得上述字符呢，这里有个简单的对照表

假设我们需要获得f这个字符就可以使用 ![]+[]来获得，其中第二个[]的效果类似于 false + ''，直接将false转成了字符串。

其中 +[]和+!+[]需要单独拎出来说明下。 我们在控制台输入+[]会得到数字0这个结果，按照之前的说法，我们尝试下+''也得到了0，那么这个结果怎么理解呢？

这里涉及到了js的隐式转换规则

  • 如果其中有一个操作数为string，则将另一个操作数隐式的转换为string，然后进行字符串拼接得出结果。
  • 如果操作数为对象({})或者是数组([])这种复杂的数据类型，那么就将两个操作数都转换为字符串，进行拼接
  • 如果操作数是像boolean这种的简单数据类型，那么就将操作数转换为number相加得出结果

+!+[]=1#

根据第二条规则 +后面跟着[]和''就是同一个效果，因为操作对象[]会被转换成''

再根据第三条规则+!+[]就会强制转换成number类型，显然结果是1

+[]=0#

1的这个结果没有问题了,那么怎么理解 +[] = 0呢？
结合知乎的参考文档，我们知道
这里看到的 + 加号并不是二元运算符「加法」，而是一个一元运算符，作用是将它后面的操作数转换成数字，和 Number() 函数完全一样。

more： 大于1的数字怎么表示#

在尝试的过程中，我试着去获得数字2， 按照上面的写法，我输入+!+!+[]得到结果0

好吧，这里又得再撸撸思路。这个!是逻辑运算符否。所以按照运算顺序来看，其实在+!+[]这个代码中，

  • 优先计算的是+[]得到0
  • 然后!0得到布尔值ture
  • + ture => Number(true) => 1

这下如果这个基础上去理解

  • +!+!+[]就等于是+!1=> 0,
  • +!!+[] => +!!0 => 0
  • +!+[]+!+[] => 可以隐藏第一个加号 ： !+[]+!+[] => 1 + 1 => 2

至此，我们回去看最上面的样例代码，就会发现很多+!+[]或者!+[]+!+[]的代码，其实就是在表示数字
